# Architectural Decision Record (ADR)

## 2026-03-30: 羅針盤（Compass/Future Lab）機能のモジュール化

### [コンテクスト]
元々の `futureLab.ts` は 1200 行を超える巨大なファイルであり、以下の役割が混在していました。
1.  **ビジネスロジック**: SEED v5 プロンプト生成、シミュレーション結果の正規化
2.  **データ集計**: アセット確定状況の集計、履歴のフィルタリング
3.  **UI 構築**: HTML 文字列の手動結合、DOM 操作、イベント登録

この「モノリス」な構造は、コードが追いづらく、画面を表示せずにロジックをテストすることが不可能であったため、機能拡張の大きな障壁となっていました。

### [決定事項]
「関心の分離（Separation of Concerns）」に基づき、アーキテクチャを 3 層に分離しました。

1.  **Logic Layer (`src/logic/compass/compassResolver.ts`)**:
    - DOM に一切依存しない純粋な TypeScript ロジック。
    - プロンプト生成、データパース、アセット検索を担当。
    - **成果**: 画面を起動せずに `Vitest` 等でロジックの正しさを検証可能。

2.  **Component Layer (`src/ui/views/compass/*.ts`)**:
    - 特定の UI 部品（世界線カード、履歴パネル等）の描画に特化した関数群。
    - `actions` オブジェクト（コールバック）を引数で受け取る疎結合な設計。
    - `sanitize` 処理を内部で完結。

3.  **Controller Layer (`src/ui/views/futureLab.ts`)**:
    - 画面全体のオーケストレーション。
    - 各コンポーネントを繋ぎ、`logic` からデータを取得し、イベント（資産確定等）を中継。

### [メリット]
- **保守性**: ファイルサイズが劇的に縮小（`futureLab.ts` 72%削減）し、修正範囲が明確。
- **堅牢性**: ロジックが独立したため、AI スキーマの変更（SEED v6 等への移行）が容易。
- **テスト可能性**: 画面上のボタンをクリックせずとも、シミュレーション処理の自動テストが可能。

### [今後のガイドライン]
新しく羅針盤に関連する機能（例：新しい解析チャート）を追加する場合は、以下のフローを推奨します。
1.  データ処理が必要な場合は `compassResolver` にロジックを追加。
2.  固有の UI 部品が必要な場合は `src/ui/views/compass/` に新コンポーネントを作成。
3.  `futureLab.ts` でそれらを統合して描画。
