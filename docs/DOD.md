# 完成定義書 (Definition of Done: DoD)

この文書は、`ippo-dashboard_online` の主要モジュールにおける機能的・非機能的な「完了」基準を定義したものです。

---

## 🏗️ 編纂室 (Henzan Room)

AIブリッジを通じて知識を抽出し、資産化（Assetize）する一連のプロセスが以下の状態にあることを「完了」とします。

### 機能要件
- [ ] **抽出・分割**: AIプロンプトにより、複雑な提案が「最小単位（Atomic）」に分割され、親子関係（Hierarchy）が自動構築されること。
- [ ] **資産化フロー**: 「採択（Adopt）」ボタン押下により、提案が永続化ストレージ（IndexedDB等）に「資産」として正しく変換・保存されること。
- [ ] **可視化**: 資産一覧がツリー構造で表示され、インデントと接続線により階層関係が直感的に理解できること。
- [ ] **詳細閲覧**: 個別の資産・提案をクリックした際に、その背景、詳細、および関連する親子資産へナビゲートできること。

---

## 🧭 羅針盤 (Compass)

未来のシミュレーションと、現在地の同期を行うコア機能が以下の状態にあることを「完了」とします。

### 機能要件
- [ ] **同期**: 「羅針盤を振る（Sync）」アクションにより、外部（OneDrive等）および内部ログの最新状態がダッシュボードに反映されること。
- [ ] **未来予測**: Ippo Log（行動履歴）に基づき、3つの世界線（Baseline, Leap, Guardrail）がChart.js上に正しくプロットされること。
- [ ] **目標連動**: 設定された目標と実際の行動指針（資産）が紐付き、優先順位が可視化されていること。

---

## 🛡️ 共通品質 (Non-Functional)

すべてのコード変更において、以下の基準を満たす必要があります。

### 品質要件
- [ ] **型安全性**: すべての新規コードにおいて `any` を排除し、適切な `interface` / `type` 定义がなされていること。
- [ ] **テスト**: `npm test` で既存の機能（特にIppo Logのパースや日付計算）が損なわれていないことを確認済みであること。
- [ ] **デザイン整合性**: Bento Grid、Glassmorphismのデザインシステムを逸脱せず、レスポンシブ（特にサイドパネルの挙動）が担保されていること。
- [ ] **セキュリティ**: ユーザー入力が `DOMPurify` 等でサニタイズされ、XSS耐性が確保されていること。
