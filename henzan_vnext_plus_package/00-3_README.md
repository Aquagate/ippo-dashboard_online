# 編纂室 集中改善パッケージ

このパッケージは、現行の `ippo-dashboard_online-1` を前提に、
**編纂室を「ルールベース抽出中心」から「AIブリッジ + 人間編集中心」へ再設計するための改善指示書** をまとめたものです。

## 同梱ファイル
- `01_改善指示書_編纂室_AIブリッジ再設計_v0.1.md`
- `02_IDEプロンプト_編纂室_AIブリッジ再設計_v0.1.md`

## 今回の判断
現行のルールベース抽出は、実運用に耐えないと判断する。

理由:
- カテゴリ頻度 → 「◯◯関連の活動」という雑な資産名が量産される
- 頻出語抽出 → 「なんか」「嬉しい」「そういえば」のようなノイズ候補が出る
- 既存のAIプロンプト生成は要約が粗く、更新・統合・昇格の判断に必要な構造が足りない
- AI取り込みが「新規候補追加」に偏っており、編纂室の本質である再編集・統合・昇格を扱えていない

## 次にやるべきこと
1. ルールベース抽出を主導線から降ろす
2. AIブリッジの入出力スキーマを新設する
3. プロンプトを「資産発見」だけでなく「更新 / 統合 / 昇格」まで扱える形にする
4. 人間が編集長として見やすい要確認トレイへ再設計する

## 方針
- AIに「候補生成」は任せる
- 人間が「採択・統合・昇格・命名」を握る
- 派手な自動化より、誤抽出を減らす
- まずは“回る編纂室”を作る
