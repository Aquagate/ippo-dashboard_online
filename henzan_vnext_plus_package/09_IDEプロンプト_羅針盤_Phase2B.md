# IDE実装プロンプト: 羅針盤 Phase 2B

あなたは、既存の `ippo-dashboard_online-1` リポジトリに対して、**羅針盤 Phase 2B** を追加改善する担当です。

## 前提
現在のリポジトリには、少なくとも以下があります。

- 一歩ログ
- Future Lab
- 編纂室
- ホビーフォートレス系の初期実装またはドキュメント
- 編纂室資産のデータ構造

ただし、現在の思想は「ホビー」よりも **羅針盤** に寄っています。  
つまり、趣味アプリやゲーム化装置ではなく、**人生の方向性を静かに示す装置** として整えたいです。

---

## 今回の目的
次の 2 点を実現してください。

1. 編纂室資産が、羅針盤の現在地・次の方向・根拠資産表示に本当に使われること
2. 時間 / 体力 / お金 / 家族といった制約資源を、軽量に表示できること

---

## 必須要件

### 1. UI名称を「羅針盤」に寄せる
- タブ名
- 見出し
- 説明文
- カードラベル

を「羅針盤」中心に調整してください。

**注意**  
内部識別子は当面 `hobbyFortress` のままでも構いません。  
壊さないことを優先してください。

### 2. 編纂室 → 羅針盤の連携を見える化する
最低限、羅針盤画面に以下を表示してください。

- 現在地
- 次の方向
- 達成済み条件
- 不足条件
- 根拠資産

### 3. 次の一歩を静かに提示する
- 1〜3 件だけ
- 命令口調にしない
- TODO 圧を強くしない
- 「候補」や「次の一歩」として穏当に出す

### 4. 制約資源の軽量表示を追加する
最低限、表示だけ追加してください。

候補:
- 時間帯: 小 / 中 / 大
- 体力帯: 守る / 進む
- お金帯: 低 / 中 / 高
- 家族制約: 軽 / 中 / 重

**注意**
- Phase 2B では深い判定ロジックに組み込まなくてよい
- まずは表示に留める
- 値は仮置き / ローカル状態 / サンプルでもよい

---

## 実装制約
- 過度なゲーム化をしない
- レベルアップ演出や派手なアニメーションは入れない
- AI自動ミッション生成は入れない
- 複数トラック対応は入れない
- 編纂室を壊さない
- Future Lab を壊さない
- OneDrive / localStorage 周辺を壊さない
- Vanilla TypeScript の流儀を維持する

---

## UX制約
以下のようなトーンを守ってください。

### 望ましい
- 現在地
- 次の方向
- あと 1 条件
- 今ある資産
- 次の候補
- 今は守る時期かもしれない

### 避ける
- クリア
- 未達成
- 失敗
- 今すぐやるべき
- レベルアップ演出
- 達成を煽る演出
