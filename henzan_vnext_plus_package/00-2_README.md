# 羅針盤 Phase 2B パッケージ

このパッケージには、次フェーズの判断と実装着手に使う文書を入れています。

- `07_改善提案書_羅針盤_Phase2B.md`
- `08_実装計画書_羅針盤_Phase2B_連携検証_v0.1.md`
- `09_IDEプロンプト_羅針盤_Phase2B.md`

## 今回の結論
IDE から出ている 5 つの候補のうち、次にやるべき本命は以下です。

1. **編纂室 → 羅針盤の連携検証**
2. **制約資源の表示反映（軽量版）**

反対に、今回は後ろに回すのが妥当です。

- 複数趣味対応
- AI自動ミッション生成
- 階層到達時の演出

## 理由
うぜんの現在の思想は、過度なゲーム化よりも、
**人生の方向性を静かに示す装置** を目指しているためです。

そのため、次フェーズは「派手さ」ではなく以下を優先します。

- 編纂室の資産が本当に羅針盤の判定に使えること
- 時間 / 体力 / お金 / 家族などの制約が見えること
- プレッシャーを生まない穏当な表示であること
- タブ名・文言・設計思想が「ホビー」から「羅針盤」に寄ること

## 補足
実装上の安全性を考え、**内部コード識別子は当面 `hobbyFortress` のままでもよい** としています。
UI 表示、文書、ラベルは「羅針盤」に寄せ、内部名は段階的に整理する方が事故が少ないためです。
