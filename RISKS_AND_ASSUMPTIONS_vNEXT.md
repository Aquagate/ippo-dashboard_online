# RISKS_AND_ASSUMPTIONS_vNEXT.md

## 既知リスク（P0実装後に更新）

### セキュリティ/プライバシー
- **保存型XSSリスク**: DOMPurifyは入っているが、`innerHTML` を使う箇所がUI層に点在。CSP導入（P2-01）まで運用注意が必要
- **ログ露出**: P0-04で `debugLogging` フラグを導入し、通常モードではMSALログがUIに流れなくなった → **リスク軽減済み**
- **同期設定情報**: clientId等がlocalStorageにある。秘密鍵ではないが、貼り付け・共有時に混入しやすい

### 運用/品質
- **データ破損時の復旧**: P0-02でlastGoodへの自動フォールバックを実装 → **リスク軽減済み**
  - ⚠️ 新リスク: 復旧後に壊れたデータが再度保存されるループの可能性。現状は `saveLastGood` が5秒後に走るため、復旧直後のIDB上書きでほぼ安全だが、完全ではない
- **テスト**: P0-03でvitest + 13テスト導入 → **リスク軽減済み**
  - ⚠️ 残存: 同期フロー（syncManager）のテストが未実施。CIで `npm test` は実行されるが、カバー範囲は限定的
- **FutureLabの肥大化**: 1200行超。改修時にバグを混入させやすい → 変更なし（P2-03で対応予定）
- **main.ts の肥大化**: 460行超の DOMContentLoaded ハンドラ → 変更なし（P1-01で対応予定）

### コスト
- **依存更新リスク**: vitest / jsdom を devDependencies に追加。CIで自動検知される体制になった → **リスク軽減済み**

### 法務
- Graph API / MSAL の利用規約は変わりなし
