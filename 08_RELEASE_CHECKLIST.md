# 08_RELEASE_CHECKLIST（RC宣言〜リリース）

## 1) 依存関係・セキュリティ
- [ ] 依存はnpm管理（CDN排除）
- [ ] `npm audit` 等の結果を確認（重大が残る場合は理由と対策）
- [ ] secretsが混入していない（.env / トークン / キー）
- [ ] CSPが最低ラインで有効（RC）
- [ ] innerHTML監査（原則ゼロ、必要箇所はサニタイズ）

## 2) 品質
- [ ] Unitテスト（merge/migration/validate/snapshots）が通る
- [ ] 最小E2E（05の1〜3本）が通る
- [ ] 主要フロー回帰なし（入力→保存→再起動、同期、復元）

## 3) 性能
- [ ] ログ件数が増えても致命的に遅くない（体感/簡易計測）
- [ ] 同期が過剰に走らない（debounce/pull間隔が妥当）

## 4) ロールバック（最重要）
- [ ] lastGoodで復元できる
- [ ] 世代スナップショットから復元できる（RC以降）
- [ ] OneDrive世代バックアップから戻せる（RC以降）
- [ ] JSONエクスポートが機能する（最終手段）

## 5) リリースノート
- [ ] 変更内容（機能/挙動/注意点）
- [ ] 既知の制限（pull間隔、同期OFF等）
- [ ] 復旧手順（復元ボタン、エクスポート/インポート）
